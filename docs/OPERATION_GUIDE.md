## 7. 节点申请管理（新增功能）

### 功能说明
管理用户提交的节点申请，包括节点合伙人（Node Partner）和节点委托人（Node Delegate）申请。

### 申请类型
| 类型 | 说明 |
|------|------|
| node_partner | 节点合伙人 |
| node_delegate | 节点委托人 |

### 申请状态
| 状态 | 说明 |
|------|------|
| pending | 待审核 |
| approved | 已通过 |
| rejected | 已拒绝 |

### 查看申请列表
1. 进入「节点申请管理」页面
2. 默认显示所有状态的申请
3. 可通过顶部筛选标签快速筛选：
   - ALL：全部
   - PENDING：待审核
   - APPROVED：已通过
   - REJECTED：已拒绝

### 申请信息
每条申请包含以下信息：
- 申请人钱包地址
- 用户昵称
- 申请类型
- 申请理由
- 联系方式
- 累计投资额
- 团队人数
- 附件链接
- 申请时间

### 审核申请（待审核状态）

**步骤：**
1. 点击待审核的申请卡片展开详情
2. 查看申请信息及附件
3. 点击底部「REVIEW」按钮
4. 在弹出框中填写审核备注（可选）
5. 选择操作：
   - **APPROVE**：通过申请
   - **REJECT**：拒绝申请
6. 确认后系统自动更新状态

**注意事项：**
- 审核通过后，用户状态将变为「approved」
- 审核拒绝后，用户状态将变为「rejected」
- 所有操作均会记录到操作日志中

## API 接口参考

### 基础信息
- Base URL: `http://localhost:9091/api/v1`
- 所有请求需在 Header 中添加 `Content-Type: application/json`

### 节点申请相关接口

#### 获取申请列表
```
GET /api/v1/node-applications
```
Query 参数：
- `page`: 页码（默认1）
- `pageSize`: 每页数量（默认20）
- `status`: 状态筛选（pending/approved/rejected）
- `apply_type`: 类型筛选（node_partner/node_delegate）

#### 获取统计数据
```
GET /api/v1/node-applications/stats
```

#### 获取申请详情
```
GET /api/v1/node-applications/:id
```

#### 创建申请
```
POST /api/v1/node-applications
```
Body 参数：
- `user_address`: 用户钱包地址（必填）
- `apply_type`: 申请类型（必填）
- `user_name`: 用户昵称
- `apply_reason`: 申请理由
- `contact_info`: 联系方式
- `total_invest`: 累计投资额
- `team_size`: 团队人数
- `attachment_url`: 附件链接

#### 审核申请
```
PUT /api/v1/node-applications/:id/review
```
Body 参数：
- `status`: 审核状态（approved/rejected）
- `reviewer_id`: 审核人ID
- `reviewer_notes`: 审核备注

## 常见问题

### Q: 登录失败怎么办？
A: 检查用户名和密码是否正确，默认测试账号为 admin/admin123

### Q: 如何重置密码？
A: 联系系统管理员通过数据库直接修改

### Q: 申请审核后用户收不到通知？
A: 当前版本不包含消息通知功能，需手动通知用户或通过链上交易确认

### Q: 如何查看某个用户的详细信息？
A: 在用户列表页面搜索钱包地址，点击用户卡片进入详情页
