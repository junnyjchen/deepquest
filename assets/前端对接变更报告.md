## 三、Vault 质押函数变更

### 3.1 数据结构变更

**旧版：** 同期限质押合并为一条记录
```solidity
// 旧版：按用户+等级存汇总数据
sAmt[user][level]  // 累计金额
stakeTime[user][level]  // 最后一次质押时间（会重置）
```

**新版：** 每笔质押独立记录
```solidity
// 新版：每笔质押独立
StakeRecord { amount, startTime, debt, active }
stakeRecords[user][level]  // 记录数组
```

### 3.2 stake 函数

| | 旧版 | 新版 |
|---|---|---|
| 函数签名 | `stake(uint8 _level, uint256 _amount)` | **不变** |
| 行为 | 金额累加，时间重置 | 新增一条独立记录 |

**前端调用方式不变。**

### 3.3 unstake 函数 ⚠️ 参数变更

| | 旧版 | 新版 |
|---|---|---|
| 函数签名 | `unstake(uint8 _level, uint256 _amount)` | `unstake(uint8 _level, uint256 _index)` |
| 第二个参数 | 解押金额 | **记录索引** |

**前端需要修改：**
- 旧版传金额，新版传记录索引
- 先调用 `getStakeRecords(user, level)` 获取记录列表
- 用户选择要解押的记录，传索引值

### 3.4 新增查询函数

| 函数 | 合约 | 说明 |
|---|---|---|
| `getStakeRecords(address user, uint8 level)` | Vault | 返回该用户指定期限的所有质押记录数组 |
| `getStakeRecordCount(address user, uint8 level)` | Vault | 返回该用户指定期限的记录数量 |

### 3.5 claimStakeReward 函数

| | 旧版 | 新版 |
|---|---|---|
| 函数签名 | `claimStakeReward(uint8 _level)` | **不变** |
| 行为 | 领取整个等级的奖励 | 遍历所有 active 记录计算总奖励 |

**前端调用方式不变。**

## 六、⚠️ 代币授权（approve）流程 [重要]

### 6.1 问题说明

部署新合约后，用户旧的所有 approve 授权全部失效（allowance = 0）。以下操作涉及代币转移，**必须在执行前检查并授权**：

### 6.2 各操作的授权要求

| 操作 | 需要授权 | 调用方法 | 是否必须 |
|---|---|---|---|
| 入金 | SOL → DQMCore | `SOL.approve(DQMCore, MAX)` | 是 |
| 授权LP权益 | 不需要 | authorizeLPEquity 只读余额 | 否 |
| **取消LP** | **LP → StakeCore** | **`DQPAIR.approve(StakeCore, MAX)`** | **是** |
| 质押DQ | DQT → Vault | `DQT.approve(Vault, MAX)` | 是 |

### 6.3 前端授权代码示例

```javascript
const MAX_UINT256 = '115792089237316195423570985008687907853269984665640564039457584007913129639935';

// 通用授权检查+执行
async function ensureAllowance(tokenContract, spender, userAddress) {
  const allowance = await tokenContract.allowance(userAddress, spender);
  if (allowance > 0n) return true; // 已授权
  const tx = await tokenContract.approve(spender, MAX_UINT256);
  await tx.wait();
  return true;
}

// ===== 各操作按钮处理 =====

// 入金
async function deposit(amountSOL) {
  await ensureAllowance(SOLContract, DQMCoreAddress, userAddress);
  await DQMCoreContract.deposit(amountSOL);
}

// 授权LP权益（同时自动approve，方便后续取消LP）
async function authorizeLP() {
  // 授权LP给StakeCore（取消LP时需要）
  await ensureAllowance(DQPAIRContract, StakeCoreAddress, userAddress);
  await StakeCoreContract.authorizeLPEquity();
}

// 取消LP
async function cancelLP(amount) {
  await ensureAllowance(DQPAIRContract, StakeCoreAddress, userAddress);
  await StakeCoreContract.cancelLPEquity(amount);
}

// 取消全部LP
async function cancelAllLP() {
  await ensureAllowance(DQPAIRContract, StakeCoreAddress, userAddress);
  await StakeCoreContract.cancelAllLPEquity();
}

// 质押DQ
async function stakeDQ(level, amount) {
  await ensureAllowance(DQTContract, VaultAddress, userAddress);
  await VaultContract.stake(level, amount);
}
```

### 6.4 建议：授权LP时一并 approve

在用户点击"授权LP"（authorizeLPEquity）时，**同时帮用户 approve LP 代币给 StakeCore**。这样用户后续取消LP时就不需要再次授权了。
