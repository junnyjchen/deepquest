## 二、前端调用地址变更

### 2.1 需要切换到新地址的合约

前端所有调用旧地址的地方，都需要替换为新地址。

### 2.2 函数调用合约变更

以下函数原来在 StakeCore 调用，**现在改到 DQMAdmin 调用**：

| 函数名 | 旧调用合约 | 新调用合约 | 参数变化 |
|---|---|---|---|
| `calculateLevel(address)` | StakeCore | **DQMAdmin** → `calculateLevelWithInfo(address)` | 返回值增加了 teamSales 和 childCount |
| `getChildCount(address)` | StakeCore | **DQMAdmin** → `getChildCount(address)` | 无变化 |
| `setLevelThresholds(...)` | StakeCore | **DQMAdmin** → `setLevelThresholds(...)` | 无变化 |

### 2.3 StakeCore 移除的函数（不要再调用）

| 函数名 | 说明 | 替代方案 |
|---|---|---|
| `calculateLevel(address)` | 已移到 DQMAdmin | 调用 `DQMAdmin.calculateLevelWithInfo(address)` |
| `getChildCount(address)` | 已移到 DQMAdmin | 调用 `DQMAdmin.getChildCount(address)` |
| `setLevelThresholds(...)` | 已移到 DQMAdmin | 调用 `DQMAdmin.setLevelThresholds(...)` |
| `batchSetDirectSales(...)` | 已移到 DQMAdmin | 调用 `DQMAdmin.batchSetDirectSales(...)` |
| `batchSetDirectCount(...)` | 已移到 DQMAdmin | 调用 `DQMAdmin.batchSetDirectCount(...)` |
| `batchSetLPStakeTime(...)` | 已移到 DQMAdmin | 调用 `DQMAdmin.batchSetLPStakeTime(...)` |

## 三、Vault 质押函数变更

### 3.1 数据结构变更

**旧版：** 同期限质押合并为一条记录
```solidity
// 旧版：按用户+等级存汇总数据
sAmt[user][level]  // 累计金额
stakeTime[user][level]  // 最后一次质押时间（会重置）
```

**新版：** 每笔质押独立记录
```solidity
// 新版：每笔质押独立
StakeRecord { amount, startTime, debt, active }
stakeRecords[user][level]  // 记录数组
```

### 3.2 stake 函数

| | 旧版 | 新版 |
|---|---|---|
| 函数签名 | `stake(uint8 _level, uint256 _amount)` | **不变** |
| 行为 | 金额累加，时间重置 | 新增一条独立记录 |

**前端调用方式不变。**

### 3.3 unstake 函数 ⚠️ 参数变更

| | 旧版 | 新版 |
|---|---|---|
| 函数签名 | `unstake(uint8 _level, uint256 _amount)` | `unstake(uint8 _level, uint256 _index)` |
| 第二个参数 | 解押金额 | **记录索引** |

**前端需要修改：**
- 旧版传金额，新版传记录索引
- 先调用 `getStakeRecords(user, level)` 获取记录列表
- 用户选择要解押的记录，传索引值

### 3.4 新增查询函数

| 函数 | 合约 | 说明 |
|---|---|---|
| `getStakeRecords(address user, uint8 level)` | Vault | 返回该用户指定期限的所有质押记录数组 |
| `getStakeRecordCount(address user, uint8 level)` | Vault | 返回该用户指定期限的记录数量 |

### 3.5 claimStakeReward 函数

| | 旧版 | 新版 |
|---|---|---|
| 函数签名 | `claimStakeReward(uint8 _level)` | **不变** |
| 行为 | 领取整个等级的奖励 | 遍历所有 active 记录计算总奖励 |

**前端调用方式不变。**

## 五、Bug 修复影响

| 修复项 | 影响 | 前端需关注 |
|---|---|---|
| 管理奖级差制 `/100→/30` | 入金奖励金额变多 | 管理奖显示金额会增加 |
| 见点奖用 directCount 判断 | 更多用户可获得见点奖 | 见点奖发放范围变大 |
| 每日入金限额重置 | 用户每天可重新入金 | 之前"限额用完"的用户可入金 |
| 爆块释放率 0.13%→1.3% | 爆块DQ数量增加10倍 | 爆块奖励显示金额变大 |
| authorizeLPEquity 无入金时用钱包LP | LP授权数量正确显示 | LP权益不再为0 |
| cancelLPEquity 移除自动发奖 | 取消LP不再报错 | 取消LP只做破LP+手续费+转币 |
| claimLPReward 更新 debt | 防止重复领取 | LP奖励领取后金额正确归零 |

## 九、前端必须修改的清单
